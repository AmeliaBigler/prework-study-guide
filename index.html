Languages

HTML

  • HTML is a coding language. HTML stands for HyperText Markup Language.
  • This language speaks to the browser.
  • The head element contains information about the webpage. This content is not visible to the user.
  • The body element represents the visible content shown to the user.
  • Tags are the code inside angled brackets. Many (but not all) elements have opening and closing tags.
  • Every visible HTML element is displayed using default style settings that come with browsers.
  • Default browser settings can be overriden by CSS. (See CSS section)

CSS

  • CSS is a coding language. CSS stands for Cascading Style Sheets.
  • This language speaks to the browser.
  • CSS applies style to HTML elements according to a model called the Box Model.
  • The 4 components of the Box Model, from the inside out, are:
    1. The content
    2. The padding
    3. The border
    4. The margin
  • Types of CSS: in-line, internal, external.
  • If using external CSS, the link must be added to the HTML file metadata / head element.

Basic Syntax

  • code { background-color: grey; }
  • (Selector) (declaration)

JavaScript

  • JavaScript is a scripting/programming language. Technically named ECMAScript.
  • ECMA stands for European Computer Manufacturers Association.
  • This language acts like a translator between browser and server, to create dynamic app functions.
  • Control Flow is the order in which a computer executes code in a script.
  • A variable is a named container that allows us to store data in our code as values.
  • Types of values include string values, number values, and boolean values. These are examples of primitive values.
  • A conditional is a logical expression that evaluates either "true" or "false".
  • Types of conditionals include == != > < === !==

Basic Syntax

  • var valueType = "string" ;
  • (var keyword) (var name) (assignment) (value) (termination)

Programs

Git

  • Git is a program used for version control of code.
  • Git Flow is the process of pulling and pushing code between the main branch and feature branches.
  • Git commands are entered in to the Command Line Interface. Frequently used git Commands:
    • git status checks what branch we are currently on.
    • git checkout -b branch-name creates a new branch and switches to it.
    • git pull origin main pulls most up-to-date base branch from remote Github in to local enivronment.

Programming Concepts

Problem Solving

  • User Story and Acceptance Criteria are used to define and navigate problems.
  • User Stories are written as "As an _, I want _, so that _." statements.
  • Acceptance Criteria are written as "GIVEN _, WHEN_, THEN_." statements.
  • A wireframe can be used to visualize the basic desired schematic of a website.

Computational Thinking

  1. Decomposition: Break the problem in to smaller parts.
  2. Pattern Recognition: Identify patterns between different parts of the problem.
  3. Abstraction: Extract the most relevant information from each part of the problem.
  4. Algorithm: Define a step-by-step solution to the problem.
